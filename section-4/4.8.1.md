# 4.8.1. “`content`”查询参数

“`content`”查询参数控制请求的数据节点的后代节点将如何在答复中处理。

允许的值是：

| 值 | 说明 |
| :------------- | :------------- |
| config | 只返回配置后代数据节点 |
| nonconfig | 只返回非配置的后代数据节点 |
| all | 返回所有的后代数据节点 |

此参数仅适用于数据存储区和数据资源上的`GET`方法。 如果用于其他方法或资源类型，则返回“`400 Bad Request`”状态行。

如果此查询参数不存在，则默认值为“`all`”。 这个查询参数必须被服务器支持。
